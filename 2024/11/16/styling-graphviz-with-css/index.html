Styling Graphviz with CSS

16 November 2024

Since I sometimes use Graphviz for articles, and now that I‚Äôm trying to make this site respect light and dark mode preferences smoothly, I wanted to get my graphs to do so as well. In the past, I‚Äôve assumed light mode only, and so my graphs look bad in dark mode ‚Äî¬†they‚Äôre a sudden flash of bright in an otherwise dark page.

To see it in action: If you‚Äôre in dark mode, switch to light, and vice-versa. (On macOS, for example, you can do this in System Settings ‚Üí Appearance.) If it‚Äôs not easy for you to switch back and forth betwixt the modes, here are screenshots:

It took me longer than it should have to figure out how to get this to work, so I thought I‚Äôd write it down. These are the key things to know:

  • Graphviz can indeed produce Scalable Vector Graphics (SVG) files. Woo hoo!
  • You can indeed style SVGs with CSS. (SVG is an application of XML, so things work as you‚Äôd hope.) As always, Mozilla Developer Network‚Äôs documentation is good.
  • You can embed SVG directly into HTML (!) ‚Äî¬†you don‚Äôt have to use an img tag.
  • To get this to work easily and consistently, you really want to do the direct embedding.

Crucial to all this is that the SVG be inline in the HTML, inside a div. I found that trying to set the stylesheet attribute in the digraph, and then loading the image in an HTML page with the img tag, resulted in Total CSS Chaos. I am nowhere near good enough at the DOM and CSS to understand what was going on; but, thankfully, putting the SVGs inline seems to fix things.

This also means you can maintain just a single CSS file for your main site; you just have to add some declarations for SVG XML elements.

Here‚Äôs the core of my CSS that gets this working:

:root {
    color-scheme: light dark;
    --fg: light-dark(#333, #ccc);
    --bg: light-dark(#fff, #333);
}

body {
    background-color: var(--bg);
    color: var(--fg);
    font-family: "Helvetica Neue";
}

.node {
    stroke: var(--fg);
    fill: var(--fg);
}

.edge {
    fill: var(--fg);
    stroke: var(--fg);
}

text {
    font-family: inherit;
    font-size: 70%;
}

.node, .edge, and text are the relevant SVG elements. The rest is for the main HTML page, but it provides variables that we‚Äôll use in the SVG-related CSS.

Here‚Äôs the Graphviz file:

digraph {
  splines = polyline
  rankdir = LR
  bgcolor = transparent
  node [
    shape = circle
    margin = 0.1
    width = 1
  ]
  edge [
    margin = 0.1
  ]

  gourd [ label = "gourd" ]
  pumpkin [ label = "pumpkin" ]
  squash [ label = "squash" ]
  decoration [ label = "decoration", shape = doublecircle ]

  gourd -> pumpkin
  pumpkin -> squash
  squash -> decoration
}
Note the bgcolor = transparent ‚Äî¬†that‚Äôs important.

To render it as an SVG, and to get the output SVG to respect our CSS, we use this pipeline:

% dot -Tsvg test.dot | sed -E -e s'/(fill|stroke)="black"/\1="var(--fg)"/g' > test.svg

Note that the -E option for sed gives us POSIX Extended regular expressions. Some sed implementations use a different option flag for Extended REs.

The sed is necessary because Graphviz inserts stroke and fill attributes into the XML elements, which override the global CSS. So, we zap ‚Äôem.

When embedding the SVG into your HTML, you need to supply alt text for accessibility. Unlike img the svg element doesn‚Äôt support the alt attribute. Instead, use role and aria-label:

<svg width="200pt" height="300pt" role="img" aria-label="Some
descriptive text goes here"
  viewBox="0.00 0.00 242.00 260.00" xmlns="http://www.w3.org/2000/svg"
  xmlns:xlink="http://www.w3.org/1999/xlink">

There‚Äôs probably an easier way to do this, but this is what I‚Äôve got so far.
